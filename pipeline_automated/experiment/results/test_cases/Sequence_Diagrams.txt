**Test Case 1: System Privilege Assignment**

* **Preconditions:** User logs in to the system
* **Step 1:** System prompts user for privilege level (admin, operator, etc.)
* **Step 2:** User selects privilege level and submits request
* **Step 3:** System assigns privileges based on user input
* **Expected Result:** User is granted assigned privileges and can access corresponding resources
* **Test Data:** Different user inputs for privilege levels (e.g. admin, operator)
* **Assumptions:**
	+ System has pre-defined privilege levels
	+ Privilege assignment is based on user's role

**Test Case 2: Access Mode Allocation**

* **Preconditions:** User requests access to critical resource
* **Step 1:** System checks if resource is available for allocation
* **Step 2:** If resource is available, system allocates resource to user and updates allocation table
* **Expected Result:** User can access allocated resource without deadlocks or errors
* **Test Data:** Critical resources with varying capacity limits (e.g. instrument slots)
* **Assumptions:**
	+ System has a dynamic allocation mechanism for critical resources
	+ Allocation table is updated in real-time

**Test Case 3: Telescope Start-up and Shutdown**

* **Preconditions:** User requests telescope start-up or shutdown
* **Step 1:** System checks if current operation allows start-up or shutdown
* **Step 2:** If allowed, system initiates start-up or shutdown sequence
* **Expected Result:** Telescope starts up or shuts down successfully without interfering with other operations
* **Test Data:** Varying operational states (e.g. telescope is running vs. idle)
* **Assumptions:**
	+ System has procedures for safe start-up and shutdown
	+ Operational state is correctly updated
