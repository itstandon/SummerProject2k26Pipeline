**Test Case ID:** OP-001 - System Privilege Establishment

**Description:**

* Verify that the system of privileges is established at each operating level of the system.
* Ensure that these privileges are determined in a simple manner during logging into the system.

**Preconditions:**

1. The user has logged into the system with valid credentials.
2. The system is in a stable state.

**Test Steps:**

1. Log into the system with an administrator account to simulate privilege establishment.
2. Verify that the system displays the established privileges for each operating level.
3. Attempt to access critical resources without going through the Access Mode Allocation system (e.g., attempt to start-up the telescope directly).
4. Check if the system denies access and prompts for proper allocation using the Access Mode Allocation system.

**Expected Results:**

1. The system displays the established privileges for each operating level.
2. Attempting to access critical resources without going through the Access Mode Allocation system is denied, and a prompt is displayed for proper allocation.
3. The system remains in a stable state with no deadlocks or errors.

**Test Case ID:** OP-002 - Access Mode Allocation System

**Description:**

* Verify that the Access Mode Allocation system dynamically identifies and assigns resources as needed.
* Ensure that critical resources are assigned solely through this allocation system.

**Preconditions:**

1. The user has logged into the system with valid credentials.
2. The system is in a stable state.
3. Critical resources are available for allocation.

**Test Steps:**

1. Simulate multiple users attempting to access critical resources at the same time.
2. Verify that the Access Mode Allocation system dynamically assigns resources as needed, preventing deadlocks or conflicts.
3. Attempt to allocate critical resources manually without using the Access Mode Allocation system (e.g., attempt to start-up the telescope directly).
4. Check if the system denies allocation and prompts for proper use of the Access Mode Allocation system.

**Expected Results:**

1. The Access Mode Allocation system dynamically assigns resources as needed, preventing deadlocks or conflicts.
2. Attempting to allocate critical resources manually without using the Access Mode Allocation system is denied, and a prompt is displayed for proper use.
3. The system remains in a stable state with no errors.

**Test Case ID:** OP-003 - Telescope Start-up and Shutdown Procedures

**Description:**

* Verify that procedures are implemented for convenient and safe telescope start-up and shutdown.
* Ensure that these procedures do not interfere with instrument operation.

**Preconditions:**

1. The user has logged into the system with valid credentials.
2. The system is in a stable state.
3. Instruments are properly configured and ready for use.

**Test Steps:**

1. Attempt to start-up the telescope using an incorrect procedure (e.g., attempting to start-up while instruments are still operational).
2. Verify that the system denies access and prompts for proper shutdown or restart procedures.
3. Simulate a normal start-up sequence using the correct procedure.
4. Check if the telescope starts up successfully without interfering with instrument operation.

**Expected Results:**

1. The system denies access and prompts for proper shutdown or restart procedures when attempting an incorrect start-up procedure.
2. The telescope starts up successfully without interfering with instrument operation when following the correct start-up sequence.

**Test Case ID:** OP-006 - Control Software Knowledge of Subsystems and Their Status

**Description:**

* Verify that the control software knows what subsystems are installed and their status at all times.
* Ensure that this information is accurately updated in real-time.

**Preconditions:**

1. The user has logged into the system with valid credentials.
2. The system is in a stable state.
3. Subsystems are properly configured and ready for use.

**Test Steps:**

1. Simulate changes to subsystems (e.g., adding or removing instruments).
2. Verify that the control software accurately updates its knowledge of installed subsystems and their status in real-time.
3. Check if the system remains stable and operational despite these changes.

**Expected Results:**

1. The control software accurately updates its knowledge of installed subsystems and their status in real-time.
2. The system remains stable and operational despite changes to subsystems.
